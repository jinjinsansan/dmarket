# 二層ポイント制 実装記録（D-market）

> 本書は **実装の確定記録**。検討用たたき台は別ファイル `二層ポイント制_設計案.md`。
> 対象実装: マイグレーション 0022〜0025（すべて本番適用済み）。作成: 2026-06-19。

---

## 0. 一言で

D-market は予測の売買を **参加ポイント**（価値ゼロ・無償発行）だけで完結させ、景品に触れるのは予測の的中報酬として得る **賞品ポイント** だけ、という**二層**構成にした。
狙いは賭博非該当の補強：「賭け」に相当する行為に財産的価値を一切介在させない。

```
[無償付与] ─→ 参加ポイント ──(売買/LMSR)──→ ポジション
  ・デイリー付与                                  │ 解決（的中）
  ・提携サイト無料特典                            ▼
                          参加ポイント払い戻し(株×100) ＋ 賞品ポイント付与(株×rate)
                                                  │
                          賞品ポイント ──(確定交換・配送先入力)──→ 景品
                                  ＼(90日で失効：日次cron)
```

---

## 1. 二つの通貨

| | 参加ポイント | 賞品ポイント |
|---|---|---|
| テーブル | `wallets` / `point_ledger`（既存・無改修） | `prize_wallets` / `prize_ledger`（0022新設） |
| 用途 | 市場の売買・LMSR・解決の払い戻し | 景品交換**専用** |
| 取得 | 無償発行（signup/daily/提携特典） | 予測の的中報酬（解決時）／ランキング報酬（将来）／管理調整 |
| 換金・譲渡 | 不可 | 不可 |
| 有効期限 | なし（退会時消滅） | 付与から **90日**（FIFOで失効） |
| 売買に使える | ○ | **×（使えない）** |

LMSR・売買RPC（0002/0004）と解決の参加pt払い戻しは**一切変更していない**。賞品ポイントは独立した報酬レイヤーとして上に載せた。

## 2. データモデル（0022）

```
prize_wallets(user_id PK, balance>=0, created_at)
prize_ledger(id, user_id, delta, reason, market_id, redemption_id, expires_at, balance_after, created_at)
  reason ∈ win_reward | rank_reward | redeem | expire | adjust
  正のdelta=付与（expires_at あり） / 負のdelta=交換・失効（expires_at null）
prizes(id, name, description, image_url, cost_points>0, stock(null=無制限), is_active, display_order, created_at)
prize_redemptions(id, user_id, prize_id, cost_points, status, shipping jsonb, created_at)
  status ∈ requested | approved | shipped | cancelled
```

**RLS**：自分の `prize_wallets`/`prize_ledger`/`prize_redemptions` のみ SELECT 可。`prizes` は `is_active=true` のみ公開。書き込みは下記 definer RPC のみ。

### 不変条件（絶対）
- `prize_wallets.balance == Σ prize_ledger.delta`（ユーザーごと）
- 残高は `grant_prize_points` / `redeem_prize` / `expire_prize_points` / 管理RPC を経由しないと変動しない。
- 参加ポイント側の不変条件 `wallets.balance == Σ point_ledger.delta` も従来どおり維持。
- いずれもローカルDBテストが常時検証（`run_local.sh`）。

## 3. RPC 一覧

### 基盤（0022・SECURITY DEFINER）
| 関数 | 役割 | 実行権限 |
|---|---|---|
| `grant_prize_points(p_user, p_amount, p_reason, p_market_id, p_expires_at)` | 賞品pt付与。台帳1行＋残高加算。期限既定 now()+90日 | サーバー専用（authenticated/anon revoke） |
| `redeem_prize(p_prize_id, p_shipping)` | 確定交換。在庫・残高を検証→残高減算＋申込作成。本人実行 | authenticated |
| `expire_prize_points()` | FIFO失効。`max(0, 期限切れ付与合計 − 消費合計)` で正確・二重失効しない | サーバー専用 |

### 的中報酬フック（0023）
`resolve_market`（0005）を**再定義**し、参加pt一括償還の直後に賞品pt付与を set-based で追加。
- 付与額 = `floor(勝ち株数 × prize_win_rate)`、有効期限 now()+90日、reason=`win_reward`、market_id 紐付け。
- レート `prize_win_rate` は `platform_settings`（既定 **1** = 勝ち1株1賞品pt）。`admin_set_setting` のホワイトリストに追加し `/admin/params` から調整可。
- `resolve_market` は status ガードで冪等 → **二重付与なし**。大量保有でも往復1回（set-based）。

### 景品・申込の管理（0024・すべて is_admin()＋_audit）
| 関数 | 役割 |
|---|---|
| `admin_list_prizes()` | 無効含む全景品（公開ページは RLS で is_active のみ） |
| `admin_upsert_prize(p_id, name, description, image_url, cost_points, stock, is_active, display_order)` | 作成（p_id=null）/更新 |
| `admin_set_prize_active(p_id, p_active)` | 公開/非公開切替 |
| `admin_list_redemptions(p_status)` | 申込一覧（景品名・申込者表示名を結合） |
| `admin_set_redemption_status(p_id, p_status)` | 発送ステータス更新。**未発送(requested/approved)→cancelled のときのみ**賞品pt返金(adjust)＋在庫復元。shipped済の取消は返金しない |

## 5. 法務文面の二層化（Phase B 同時）

賭博非該当の論拠を二層に合わせて補強（機械置換ではなく文意を保った改稿）。
- `legal/terms`：第2条で参加/賞品ポイントを分離定義＋総称を定義。第4条を二層構成、第5条「的中時に賞品pt付与」、第6条は賞品ptで景品交換。
- `legal/no-gambling`：「賭けに相当する売買に用いるのは財産的価値のない参加ポイントだけ／景品に触れるのは的中報酬の賞品ポイントのみ」を明示。
- `legal/privacy` / `legal/tokushoho`：取得情報・利用目的・料金/返還を二層に整合。
- `components/MarketDetailClient`：解決ルール文の旧コピー（「得られるのは称号とランキングだけ」）を賞品pt→景品の導線に修正。

> ⚠️ 文面は**公開前に弁護士レビュー必須**（景表法の景品最高額/総額規制ほか）。特商法の会社情報は【要記入】のまま。

## 6. 失効cron（0025・Phase D・リモート専用）

`expire_prize_points()` は純plpgsql（Edge Function/秘密鍵不要）なので、`refresh-stats` と同様に pg_cron から直接実行。
- ジョブ名 `expire-prize-points`、スケジュール `0 19 * * *`（= **04:00 JST**）、active。
- pg_cron は 0009 で有効化済み。本番で `cron.job` 登録を確認済み。

## 8. 運用パラメータ

| キー | 既定 | 調整 | 効果 |
|---|---|---|---|
| `prize_win_rate` | 1 | `/admin/params` | 的中報酬 = floor(勝ち株数 × rate)。景品コスト割れ防止のため低めから |
