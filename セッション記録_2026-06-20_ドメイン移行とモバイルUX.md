## 1. 独自ドメイン移行（dmarket-six.vercel.app → d-market.io）

コードは `NEXT_PUBLIC_SITE_URL` に集約済みで**旧ドメインのハードコードは無し**。移行は env 中心。

### 外部設定（オーナー側で適用済み）
1. Vercel → Domains に `d-market.io` 追加・DNS 向け（apex/www の正を統一）
2. Vercel 環境変数 `NEXT_PUBLIC_SITE_URL = https://d-market.io`（末尾スラッシュ・空白なし）→ 再デプロイ
3. LINE Developers の Callback URL に `https://d-market.io/api/auth/line/callback` を追加（exact一致必須）
4. Supabase → Auth → URL Configuration に新ドメイン追加

> ⚠️ #2 env / #1 着地ホスト / #3 LINE Callback の **3ホストを完全一致**させること（不一致だと LINE が redirect_uri で弾く）。

### コード/ドキュメント（コミット `9555b8a`）

- `.env.example` / `引継ぎ書`: URL 表記を新ドメインへ更新。

## 3. モバイルUX刷新

### 3.1 取引バーのフッター被り＋シートのがくつき（コミット `4676c11`）
- 市場詳細の取引バーを `bottom-16`(64px固定) → `bottom: calc(3.5rem + env(safe-area-inset-bottom))`。ホームバー安全領域を考慮し BottomNav の上へ。
- ボトムシートのがくつき対策: 取引バーの `backdrop-blur` を solid 化＋**シート表示中はバーを非表示**（背後の backdrop-filter 再合成を排除）。シートを「外側=transformアニメ／内側=スクロール」の2層に分離。表示中は背景スクロールロック＋`overscroll-contain`。シート下端に safe-area パディング。

### 3.2 固定フッターが浮く → アプリシェル構成（コミット `370546f` 試行 → `a24c9ee` 確定）
- 問題: ドキュメント全体スクロールでは「弾む滑らかさ（ラバーバンド）」と「固定フッターが浮かない」が両立しない。`overscroll-behavior:none` は浮きは止めるが滑らかさを消す（`370546f` で確認 → 撤回）。
- 解決（**アプリシェル**・`a24c9ee`）:
  - `body` を `100dvh` 固定・`overflow:hidden`、`main(#app-scroll)` だけをスクロール。
  - **TopNav/BottomNav をスクロール領域の外**に配置 → `position:fixed` 廃止で構造的に常時固定（浮かない）。
  - `main` は `overscroll-behavior-y:contain` ＋ momentum で**弾む滑らかさを維持**。
  - `Footer` を `main` 内へ移動。ページ遷移の先頭戻しに `components/ScrollReset.tsx` を追加（`#app-scroll` を scrollTo(0,0)）。
  - 詳細ページ: シートのスクロールロック対象を `#app-scroll` に、PCトレードパネルの sticky を `top-[88px]→top-4` に調整。
  - 結果: フッター固定 ＋ 弾むスクロールを両立（ユーザー確認OK）。

### 3.3 ボタンの見た目＋表示ラグ（コミット `7825b4b`）
- UI: マイページのプロフィールヘッダーを折り返し対応・ランクバッジを `whitespace-nowrap` のクリーンなピル・受取ボタンをモバイル全幅。ランキングの「総資産/的中率」をモバイル全幅セグメント＋タップ領域拡大。
- ラグ短縮:
  - `mypage / prizes / TopNav / TradePanel` の `auth.getUser()`（毎回ネットワークでJWT検証）→ **`getSession()`**（ローカルキャッシュ・ネットワーク不要）。
  - `leaderboard`(force-dynamic)/`mypage`/`prizes` に `loading.tsx` スケルトンを追加（タップ即時フィードバック）。

## 4. 重要メモ（次セッション向け）

- **スクロールは window ではなく `#app-scroll`**（アプリシェル）。スクロール位置を扱うコードは `#app-scroll` を対象にすること。新規フルスクリーンのオーバーレイ/シートは `position:fixed`（viewport基準）でOK。
- 残ラグは Supabase クエリ応答が主因。さらに詰めるならクエリ絞り込み/キャッシュ導入。
- 二層ポイント制の運用・公開前項目（特商法の会社情報【要記入】＝**後回し中**、弁護士レビュー、配送オペ）は未了。
