# ゴリラ予想 デザインブリーフ（Claudeデザイン向け）

> 「ゴリラ予想」のUI/UXリデザイン依頼書。白基調ミニマル＋ポップ、Xシェア最適化、コメントが活発になるUI。
> 作成: 2026-06-29 ／ 前提: Next.js 16 (App Router) + Tailwind v4 + Recharts。既存トークンは `web/app/globals.css`。
> 全体方針: `ゴリラ予想_リニューアル計画_2026-06-29.md`

---

## 1. プロダクト概要（デザイナー向け1分説明）

- **何**: 換金不可ポイントで遊ぶ予測市場（Polymarket型の「価格＝確率がトレードで動く」を、賭けではない遊びとして再現）。
- **誰に**: 予測市場をやったことがない**初心者・一般層**。むずかしくなく、SNSで盛り上がる体験。
- **トーン**: 親しみやすい・ポップ・でも上品（白基調でミニマル）。マスコットは**ゴリラ**。
- **やらないこと**: ギャンブルっぽい煽り、カジノ的な装飾、換金の訴求。あくまで「みんなで予想して楽しむ」。

---

## 2. ブランド

- 名称: **ゴリラ予想**（英字併記する場合の案: GORI / Gorilla Yosō）。
- マスコット: **シンプルでフラットなゴリラ**（線少なめ・絵文字的に使える）。用途＝ロゴ、空状態、ローディング、的中演出、ランク/称号アイコン、OGP。
- 性格: 賢くて頼れるが、ちょっとお茶目（バナナ＝アクセントのモチーフ）。

---

## 3. デザイン原則

1. **白基調・余白たっぷり**（情報の呼吸。カードは白、背景はオフホワイト）。
2. **ミニマル＋1〜2色のポップアクセント**（色を絞る。アクセントはCTA・強調・ブランドのみ）。
3. **角丸・やわらかい影**（既存 `--radius:20px` を踏襲。親しみ）。
4. **数字が主役**（確率%・倍率・ポイントを大きく明快に。等幅フォントで桁を安定）。
5. **モバイルファースト**（アプリシェル構成。下部 BottomNav は既存）。
6. **シェアしたくなる**（市場カード・的中結果が「画像で映える」）。

## 4. カラートークン（提案＝差替可。値だけ変えれば全体反映）

既存の変数名はそのまま、値を白基調＋ポップへ。**アクセントは2案から選定**。

```css
:root {
  --bg:        #fbfbf9;  /* 温かいオフホワイト（純白より目に優しい） */
  --surface:   #ffffff;
  --surface2:  #f4f4f1;
  --border:    #ece9e3;
  --text:      #1a1714;  /* 温かみのあるインク（真っ黒を避ける） */
  --dim:       #6b6660;
  --faint:     #a8a39b;

  /* ▼ ブランド/CTAアクセント：A案かB案で選定 */
  /* A案「バナナ＆ジャングル」: 元気でゴリラらしい */
  --primary:      #16a34a;  /* ジャングルグリーン（CTA・ブランド） */
  --primary-weak: #e8f6ec;
  --accent2:      #ffc400;  /* バナナイエロー（ハイライト・装飾） */
  /* B案「グレープ・ポップ」: YES/NOの緑赤と色がぶつからない */
  /* --primary:      #6c5ce7;  グレープ・バイオレット
     --primary-weak: #efeefe;
     --accent2:      #ffc400;  バナナイエロー */

  /* 予想の YES/NO（上げ/下げ）。ブランド色とは必ず区別する */
  --pos:      #16b364;  /* YES＝上がる（グリーン） */
  --pos-weak: #e6f7ee;
  --neg:      #ff5a5f;  /* NO＝下がる（コーラルレッド） */
  --neg-weak: #ffecec;

  --radius: 20px; --radius-sm: 12px;
  --shadow: 0 1px 2px rgba(26,23,20,.04), 0 14px 40px -18px rgba(26,23,20,.14);
}
```

> 注意: **A案はブランド緑とYES緑が近い** → 採用時はブランド緑を濃いめ＋YES緑を明るめにするか、CTAは緑・YESは別トーンで差別化。色のぶつかりを避けたいなら**B案（グレープ）推奨**。最終はデザイナー判断。

- フォント: 本文＝丸み寄りのサンセリフ（Noto Sans JP＋Inter系）。見出し＝**少しポップな丸ゴシック/ディスプレイ**で「ゴリラ予想」らしさ。数字＝等幅。
- ダークモード（`html.dark`）も同トーンで用意。

## 5. 画面・コンポーネント別の要件

### 5.1 市場カード（一覧／MarketCard）
- 大きな**確率%**＋小さなスパークライン（既存 Sparkline）＋カテゴリチップ＋締切。
- 初心者向けに**わかりやすい一文**（「〇〇は起きる？」）。YES/NO の色を明快に。
- 右上に**シェアボタン**（Xへ。カードが画像化される前提）。

### 5.2 市場詳細＋トレードパネル（MarketDetailClient / TradePanel）
- 上部に確率チャート（Recharts・期間切替 1H/6H/1D/1W/ALL は既存）。
- 「YESを買う／NOを買う」を**初心者語**で（例: 「起きる方に乗る／起きない方に乗る」）。約定後の確率プレビュー。
- 残高は**参加ポイント**表記。賞品ポイント残は別表示（既存二層）。
- **コメント欄（5.5）を目立つ位置に**。

### 5.3 「貯める」ページ（タブ統合）
ポイントを稼ぐ導線を1ページに。タブ例: **ミッション / 友達紹介 / シェア / 案件（オファーウォール） / ランキング**。
- ログインボーナス（連続日数の演出）。
- **シェアボーナス**（自分の予想をXへ→付与）。
- **友達紹介**（招待コード/リンク、紹介人数の可視化）。
- **乗っかり報酬の説明**（自分のシェア経由で誰かが乗って的中すると、的中分の1%が参加ptで入る）。
- 案件（オファーウォール）枠（将来。`参加ポイント獲得` 参照）。
- すべて「**もらえるのは参加ポイント＝換金不可**」を明示（誤解防止＋法務）。

### 5.4 Xシェア / OGP（⑦）
- **動的OGP画像**: 市場名＋現在の確率（YES%）＋ゴリラ＋ブランド。的中後は「的中！」バージョン。
- シェア定型文（市場名＋自分の予想＋リンク＋ハッシュタグ）。
- ワンタップでXへ。**シェア→戻ってきてボーナス受領**の動線をスムーズに。

### 5.5 コメント（⑤b・活発化が目的）
- 市場ごとのスレッド。**いいね・返信**。
- 任意で「この人はYESに乗ってる」等の**ポジションバッジ**（盛り上げ）。
- **通報ボタン**＋NG表示（モデレーション前提）。
- 空状態に「最初のひと声を！」とゴリラ。

### 5.6 ユーザー作成市場（⑥）
- 「市場を作る」入口（**審査通過ユーザーのみ**活性。未承認は申請導線）。
- 作成フォーム（テーマ・選択肢・締切・カテゴリ・解決ルール）。**解決は管理者**である旨を明示。
- 作成者ダッシュボードに**テラ銭（参加pt・換金不可）**の獲得状況。

### 5.7 共通
- 空/ローディング/エラー状態に**ゴリラのイラスト**。
- 称号・ランキングはポップに（賞品ゼロ＝称号が主役、既存方針）。
- BottomNav（モバイル）/ TopNav（PC）は既存構成を踏襲しつつ刷新。

## 6. やってほしいこと（Claudeデザインへの依頼）

1. 上記トークン（A/B案）から**配色を確定**し、ライト/ダークのパレットを提示。
2. **ゴリラのマスコット**（ロゴ＋数パターンの表情/ポーズ）。
3. 主要画面のモック: ①一覧 ②市場詳細＋トレード ③貯める ④コメント ⑤OGP画像 ⑥ユーザー作成。
4. コンポーネント仕様（カード・ボタン・チップ・タブ・バッジ・トースト）。
5. **Xで映える**カード/OGPのビジュアル。

## 7. 制約・トーンの厳守
- **ギャンブル/カジノ的表現はNG**（賭博非該当の方針）。「賭ける」より「予想する・乗る」。
- **換金の訴求はしない**（もらえるのは換金不可ポイント）。
- 初心者がひと目で「何を当てるゲームか」分かること。
